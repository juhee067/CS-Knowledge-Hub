# CS Knowledge Hub

> CS 팀의 흩어진 지식을 한 곳에 모아, **누가 답하든 같은 품질**을 만드는 사내 지식 자산화 플랫폼

---

## 왜 만들었나요?

고객 문의에 답할 때 이런 경험, 있지 않으신가요?

- 같은 질문인데 담당자마다 답이 다르다
- 클라이언트별 설정이 달라서 잘못 안내했다가 재문의가 온다
- 베테랑한테 물어봐야 알 수 있는 암묵지가 너무 많다
- 신규 입사자가 "어디서 찾아야 하지?" 를 먼저 찾는다

**CS Knowledge Hub**는 이 문제를 해결하기 위해 만든 도구입니다.  
지식이 없는 게 아니라, **찾을 경로가 없는 것**이 문제라는 가설에서 출발했습니다.

---

## 핵심 기능

### 🔍 통합 검색

FAQ와 클라이언트 설정 카드를 한 곳에서 검색합니다.  
키워드 검색과 의미 기반(AI) 검색을 함께 사용해, "정확히 기억 못해도" 찾을 수 있습니다.

### 📋 FAQ 관리

- 질문/답변을 마크다운으로 작성하고 카테고리·태그로 분류
- `draft → verified → deprecated` 상태 관리로 신뢰할 수 있는 답변만 상위 노출
- 누가 언제 무엇을 바꿨는지 버전 이력 보관

### 🏢 클라이언트별 설정 카드

삼성, 부산대처럼 클라이언트마다 다른 설정·예외 규칙을 카드 한 장에 정리합니다.  
답변 작성 화면에서 해당 클라이언트의 override 규칙이 자동으로 경고 배너로 표시됩니다.

### 🤖 Ask the Hub (AI 답변 어시스턴트)

자연어로 질문하면 과거 문의, FAQ, 클라이언트 설정 카드를 근거로 답변 초안을 만들어 줍니다.

```
담당자: "A 고객이 결제가 이중으로 빠졌다는데 어떻게 답하죠?"

Hub:  과거 유사 문의(#1043)에서는 이렇게 답했습니다.
      ── 근거 ──
      • [문의 #1043] B 고객 이중결제 답변 (현우)
      • [FAQ] 결제 오류 시 대응 절차 (verified)
      • [클라이언트 카드] A: 환불은 본사 승인 필수 ⚠

      ✅ 추천 답변 초안: ...
      [이 초안 복사]  [FAQ로 저장]  [근거가 부족해요]
```

> 검색된 근거에만 기반해 답변을 만들며, 근거가 없으면 임의로 추측하지 않습니다.

### 📥 데이터 수집 파이프라인 (Intake Pipeline)

문의가 어디서 오든 자동으로 Hub에 쌓입니다.

| 채널           | 방식                              |
| -------------- | --------------------------------- |
| Google Forms   | Apps Script 웹훅 → 자동 적재      |
| 이메일 (Gmail) | Gmail API Pub/Sub → 자동 적재     |
| Slack          | Events API → 자동 적재            |
| 카카오톡 채널  | 비즈 채널 웹훅 → 자동 적재        |
| SMS            | Solapi 수신 웹훅 → 자동 적재      |
| 전화 / 기타    | 담당자 단축 입력 UI               |
| 과거 데이터    | CSV/Excel 업로드, 위키 URL 임포트 |

### ♻️ 자가 적립식 지식 성장

별도의 문서 작업 없이 답변 흐름 안에서 지식이 쌓입니다.

- 답변 작성 후 **"FAQ로 저장"** 한 번으로 자산화
- Ask the Hub 답변도 바로 FAQ로 저장 가능
- 자주 묻지만 FAQ가 없는 문의는 자동으로 **자산화 큐**에 등록

| ------------ | ------------------------------------------- |
| 프론트엔드   | React 19 + TypeScript + Vite + Tailwind CSS |
| UI 컴포넌트  | shadcn/ui                                   |
| 백엔드       | Supabase Edge Functions (Deno)              |

## 사용자 권한

| 역할     | 할 수 있는 것                                         |
| -------- | ----------------------------------------------------- |
| `viewer` | 검색·조회                                             |
| `editor` | FAQ·클라이언트 카드 작성 및 수정, 문의 처리           |
| `lead`   | 검증(verified 전환)·삭제·자산화 큐 관리·대시보드 조회 |

## 출시 계획

```
Phase 0  데이터 시드       과거 FAQ·문의 CSV 이관, 클라이언트 카드 초기 세팅
   │
Phase 1  지식 기반 (MVP)   FAQ 관리, 클라이언트 카드, 통합 검색, 권한
   │
Phase 2  자동화            채널 커넥터 연동, 자동 분류·추천, Ask the Hub
   │
Phase 3  품질 루프         피드백 학습, 노후화 알림, 전화 STT 자동화
```

## 시작하기 (개발 환경)

### 1. 의존성 설치

```bash
npm install
```

### 2. Supabase 설정

```bash
# 로컬 스택 실행 (Docker 필요) — 마이그레이션·시드 자동 적용
supabase start

# 또는 Supabase Cloud 프로젝트를 사용하는 경우
supabase link --project-ref <your-project-ref>
supabase db push        # 마이그레이션 적용
```

마이그레이션·시드는 [`supabase/`](./supabase/) 에 있습니다.

- `migrations/` — 스키마(`pgvector`), 트리거(프로필 자동 생성·FAQ 버전), RLS 정책
- `seed.sql` — 로컬 개발용 샘플 클라이언트·FAQ

### 3. 환경변수

```bash
cp .env.example .env.local
# .env.local 에 VITE_SUPABASE_URL / VITE_SUPABASE_ANON_KEY 입력
# (supabase start 출력값 또는 Cloud 대시보드 → Project Settings → API)
```

### 4. 개발 서버

```bash
npm run dev      # http://localhost:5173
npm run build    # 타입체크 + 프로덕션 빌드
npm run lint     # ESLint
```

### 권한 참고

첫 회원가입자는 자동으로 `lead`, 이후 가입자는 `editor` 가 됩니다(트리거). 역할은 `lead` 가 조정합니다.
